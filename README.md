Тестовое задание: Система аутентификации и разрешений

Этот проект — реализация кастомной системы доступа на Django. Основная идея была в том, чтобы сделать проверку прав максимально гибкой, не ограничиваясь стандартными группами Django.

Как запустить проект:

1. Создать виртуальное окружение: python -m venv venv
2. Активировать его: .\venv\Scripts\activate
3. Поставить зависимости: pip install -r requirements.txt
4. Применить миграции: python manage.py migrate
5. Загрузить тестовые данные: python manage.py seed_data
6. Запустить сервер: python manage.py runserver

После этого API будет доступно по адресу http://127.0.0.1:8000/api/docs/ (Swagger).

Как работают права доступа:

Я реализовал трехуровневую проверку в файле permissions_system/services.py. Сначала система проверяет, есть ли доступ к конкретному объекту (например, документу), потом смотрит личные права пользователя, и в самом конце — права его роли. Это позволяет очень детально настраивать, кто и что может делать.

Функционал пользователей:

- Регистрация и вход через email.
- Работа через JWT токены.
- Мягкое удаление (is_active=False). Если удалить аккаунт, пользователь больше не сможет зайти, но данные останутся в базе.

Почему сделано именно так:

Стандартные возможности Django не позволяют легко давать доступ к конкретным записям в базе (например, разрешить юзеру редактировать только один конкретный проект). Моя система эту проблему решает.
